관심이 생긴 이후 게임을 만들거나 웹페이지를 제작하는 등 다양한 시도를 해보았지만, 수많은 오류를 경험하며 기초가 부족하다는 점을 느꼈고 기초를 탄탄하게 다져야 한다고 생각했습니다.

[ 대학교 ]

대학교에 입학한 이후에는 고등학교 때 경험하지 못했던 다양한 프로젝트를 진행하며 프론트엔드와 디자인에 대해 더욱 깊이 배우게 되었습니다.

처음 진행했던 프로젝트인 LANssanger에서는 프론트엔드와 디자인을 담당하였습니다.

GitHub 협업 방식과 Qt Creator를 사용한 UI 구현을 처음 경험할 수 있었던 의미 있는 프로젝트였습니다.

메신저와 CPU의 감성을 결합한 인터페이스를 구성하고, 사용자와 상호작용하는 화면을 설계하며 개선해가는 과정은 저에게 큰 흥미와 동기를 주었습니다.

[ 가장 영향을 끼친 사건 ]

LANssanger 프로젝트를 진행하며 디자인했던 화면과 실제 구현 결과가 다르게 출력되는 문제를 경험했습니다.

버튼 위치가 의도한 대로 정렬되지 않거나 해상도에 따라 화면 비율이 깨지는 문제가 있었고, 프론트엔드와 백엔드를 연결하는 과정에서는 메시지 송수신 기능이 정상적으로 동작하지 않거나 실시간 데이터가 화면에 바로 반영되지 않는 문제도 발생했습니다.

처음에는 오류가 발생하는 원인을 찾기 어려웠지만, 팀원들과 함께 기능 흐름을 단계별로 분석하고 오류 로그를 확인하면서 메시지 송수신 과정을 반복적으로 테스트했습니다.

그 결과 데이터 전달 과정에서 일부 정보가 정상적으로 처리되지 않는 부분을 발견할 수 있었고, 백엔드를 담당한 팀원과 함께 데이터 구조와 화면 출력 방식을 다시 분석하며 프론트엔드와 백엔드 간 데이터 전달 방식을 수정했습니다.

이후 기능별 작업 영역을 명확하게 나누고 GitHub 브랜치를 분리하여 작업하였으며, 커밋 메시지에 작업 내용과 변경 사항을 구체적으로 기록하며 진행 상황을 공유했습니다.

그 과정에서 충돌 발생 횟수를 줄일 수 있었고, 작업 과정 또한 이전보다 원활하게 진행할 수 있었습니다.

이 경험을 통해 단순히 기능을 구현하는 능력뿐만 아니라 문제가 발생했을 때 팀원들과 함께 원인을 분석하고 해결하는 협업과 소통의 중요성을 배울 수 있었습니다.

또한 프론트엔드 개발자는 단순히 화면만 구현하는 사람이 아니라 디자인과 백엔드 구조, 사용자 경험(UI/UX)까지 함께 이해하며 프로젝트 전체 흐름을 고려해야 한다는 점을 깊이 깨달았습니다.

[ 현재 진행중인 작업 ]

현재는 캡스톤 프로젝트 Medical Script를 진행하고 있습니다.

당뇨 환자들을 위한 애플리케이션으로, React Native 기반 모바일 프론트엔드를 담당하고 있습니다.

처음 React와 Figma를 사용해 프로젝트를 진행하다 보니 컴포넌트 구조 설계, 상태 관리, 디자인 구현, 백엔드 연동 등 여러 부분에서 부족함을 느꼈습니다.

특히 백엔드 API를 연결하는 것과 제가 만든 디자인을 React Native와 Node.js를 사용해 구현하는 과정에서 어떤 데이터를 어디에 사용해야 하는지 헷갈리는 경우가 많았고, 디자인 레이아웃이 깨지는 오류도 경험했습니다.

이를 해결하기 위해 React 구조 설계와 상태 관리 방식을 별도로 학습하며 컴포넌트를 정리하는 연습을 진행했습니다.

또한 React Native 공식 문서와 다양한 개발 자료를 참고하며 Node.js와의 연동 방식을 학습하고 직접 예제를 찾아보며 연습했습니다.

API 연동 과정에서는 단순히 기능 구현에만 집중하는 것이 아니라 API 명세서를 직접 분석하며 어떤 데이터가 전달되고 사용되는지 흐름을 정리했습니다.

이후 필요한 데이터를 화면 구조와 연결하고 반복적으로 테스트하며 데이터 처리 과정을 개선해 나가고 있습니다.

디자인 구현 과정에서는 Figma에서 설계한 화면과 실제 모바일 환경 사이의 차이를 확인하기 위해 여러 기기 크기를 기준으로 테스트를 진행했습니다.

특히 버튼 크기와 정보 배치, 입력 흐름 등을 반복적으로 수정하며 인터페이스를 구성하고 있습니다.

프로젝트를 진행할수록 코드 구조와 유지보수성, 그리고 사용자 경험(UI/UX)의 중요성을 느끼게 되었습니다.

[ 현재와 미래 ]

현재는 이러한 경험을 바탕으로 백엔드 API를 활용한 화면 구현과 전체 UI 구성 작업을 진행하고 있으며, React Native를 사용해 모바일 환경에 맞는 사용자 흐름과 인터페이스를 프로젝트에 적용하고 있습니다.

개발은 단순히 각자의 기능만 구현하는 것이 아니라 서로의 역할과 구조를 이해하며 함께 완성해가는 과정이라는 점을 배울 수 있었습니다.

특히 이번 프로젝트에서는 프론트엔드와 백엔드가 각자 역할을 나누어 작업하며 단순한 화면 구현뿐만 아니라 API 연결 방식과 데이터 흐름까지 함께 이해할 수 있었습니다.

단순히 보기 좋은 화면을 만드는 것에 그치지 않고, 디자인과 기능성을 함께 고려하여 사용자가 쉽고 편리하게 사용할 수 있는 인터페이스를 설계하는 프론트엔드 개발자로 성장하고 싶습니다.

또한 다양한 프로젝트와 협업 경험을 통해 기술적인 성장뿐만 아니라 소통 능력과 협업 능력까지 함께 발전시키며, 사용자에게 실질적인 도움을 줄 수 있는 사용자 중심 서비스를 만드는 프론트엔드 개발자가 되는 것이 가장 큰 목표입니다.

ARTICLE.TXT

생성형 UI(Generative UI)의 확산

"모두에게 같은 화면을 보여주던 시대는 끝났습니다"

최근 AI 기술의 발전과 함께 IT 업계에서는 생성형 UI(Generative UI)가 새로운 인터페이스 패러다임으로 주목받고 있습니다.

지금까지의 UI는 개발자가 미리 설계한 화면을 사용자에게 제공하는 방식이었습니다. 하지만 생성형 UI는 사용자의 의도와 상황을 AI가 분석하여 필요한 화면을 실시간으로 만들어낸다는 점에서 기존 UI와 큰 차이를 가지고 있습니다.

인류의 UI는 명령어 기반의 CLI, 그래픽 기반의 GUI, 모바일 터치 UI를 거쳐 발전해 왔으며, 생성형 AI의 등장과 함께 네 번째 UI 패러다임으로 변화하고 있다는 평가를 받고 있습니다.

━━━━━━━━━━━━━━━━━━━━━━

생성형 UI가 주목받는 가장 큰 이유는 빠른 제작 속도와 높은 생산성 때문입니다.

기존에는 하나의 화면을 제작하기 위해 기획, 디자인, 개발 과정을 순차적으로 진행해야 했지만, 생성형 UI는 사용자의 요구사항만으로도 다양한 화면 시안을 빠르게 생성할 수 있습니다.

또한 반복적인 UI 제작 작업을 줄여 개발자와 디자이너가 더 중요한 문제 해결과 사용자 경험 설계에 집중할 수 있도록 돕고 있습니다.

━━━━━━━━━━━━━━━━━━━━━━

생성형 UI의 가장 큰 특징은 사용자의 상황에 따라 인터페이스가 실시간으로 변화한다는 점입니다.

필요한 기능만 화면에 제공하여 정보 과부하를 줄이고, 사용자의 목적에 맞는 인터페이스를 즉시 생성할 수 있습니다.

ACM DIS 2025 연구에서는 이를 "선택 중심 인터페이스에서 생성 중심 인터페이스로의 변화"라고 설명하며 앞으로의 UI는 고정된 화면보다 유동적으로 변화하는 구조가 될 것이라고 전망하고 있습니다.

━━━━━━━━━━━━━━━━━━━━━━

대표적인 생성형 UI 사례는 다음과 같습니다.

  • Figma AI : 텍스트 기반 UI 시안 생성
  • Google Stitch : 자연어 기반 UI 및 코드 생성
  • Cursor : AI 코드 생성 및 UI 구현 지원
  • GitHub Copilot : 코드 자동 추천 및 생산성 향상
━━━━━━━━━━━━━━━━━━━━━━

하지만 생성형 UI가 완벽한 것은 아닙니다.

AI가 사용자의 의도를 잘못 해석하거나 부정확한 정보를 기반으로 인터페이스를 생성하는 경우가 존재합니다.

또한 접근성과 사용자 경험을 충분히 고려하지 못하는 사례도 보고되고 있으며, 생성된 결과물의 품질이 일정하지 않다는 문제도 있습니다.

현재 연구에서는 생성형 UI에 대한 설계 기준과 평가 기준이 아직 충분히 정립되지 않았다는 점을 한계로 지적하고 있습니다.

━━━━━━━━━━━━━━━━━━━━━━

저는 생성형 UI 기술이 앞으로 디자인과 프론트엔드 개발 방식을 크게 변화시킬 것이라고 생각합니다.

누구나 AI를 통해 화면을 만들 수 있는 시대가 된다면 중요한 것은 단순히 UI를 생성하는 능력이 아니라 사용자를 얼마나 이해하고 더 나은 경험을 제공할 수 있는가라고 생각합니다.

AI는 효율적인 결과물을 빠르게 만들어낼 수 있지만, 서비스의 방향성과 사용자 경험을 결정하는 것은 결국 사람의 역할이라고 생각합니다.

따라서 앞으로의 프론트엔드 개발자는 AI를 활용하는 능력과 함께 사용자 경험(UI/UX)을 설계하는 역량이 더욱 중요해질 것이라고 생각합니다.

React

사용자 인터페이스를 효율적으로 구축하기 위한 JavaScript 라이브러리

최근 가장 관심 있게 공부하고 있는 기술은 React입니다.

React는 Meta(Facebook)에서 개발한 JavaScript 기반 UI 라이브러리로, 사용자 인터페이스를 효율적으로 구성하기 위해 사용됩니다.

특히 Component 기반 구조를 사용하기 때문에 버튼, 카드, 입력창과 같은 UI 요소를 독립적인 단위로 분리하여 개발할 수 있으며, 높은 재사용성과 유지보수성을 제공합니다.

━━━━━━━━━━━━━━━━━━━━━━

► React를 사용하는 이유

HTML, CSS, JavaScript만으로도 웹페이지를 제작할 수 있지만, 서비스 규모가 커질수록 코드의 관리와 유지보수가 어려워질 수 있습니다.

React는 Virtual DOM을 활용하여 변경된 부분만 효율적으로 업데이트하기 때문에 불필요한 렌더링을 줄일 수 있으며, 보다 빠르고 부드러운 사용자 경험(UI/UX)을 제공합니다.

또한 컴포넌트 단위로 화면을 구성하기 때문에 반복되는 UI를 쉽게 재사용할 수 있고, 대규모 프로젝트에서도 일관된 구조를 유지할 수 있습니다.

► Component Based Architecture

React의 가장 큰 특징은 컴포넌트 기반 구조입니다.

하나의 화면을 여러 개의 독립적인 컴포넌트로 나누어 구성하며, 이를 조합하여 하나의 페이지를 완성합니다.

컴포넌트 단위로 개발하기 때문에 기능별 관리가 가능하며, 유지보수와 확장이 용이하다는 장점을 가지고 있습니다.

► Virtual DOM

React는 Virtual DOM을 사용하여 실제 DOM의 변경 사항을 최소화합니다.

전체 화면을 다시 그리는 것이 아니라 변경된 부분만 찾아 업데이트하기 때문에 빠른 렌더링 성능을 제공합니다.

이를 통해 사용자와 상호작용이 많은 서비스에서도 안정적인 인터페이스를 구현할 수 있습니다.

━━━━━━━━━━━━━━━━━━━━━━

► JSX

React는 JSX(JavaScript XML)라는 문법을 사용합니다.

JavaScript 코드 안에서 UI 구조를 함께 작성할 수 있어 화면 구조와 동작 로직을 하나의 컴포넌트 안에서 관리할 수 있습니다.

또한 코드의 가독성을 높여주고 복잡한 인터페이스를 보다 직관적으로 구성할 수 있도록 지원합니다.

━━━━━━━━━━━━━━━━━━━━━━

► React가 활용되는 이유

현재 React는 가장 널리 사용되는 프론트엔드 기술 중 하나로 자리 잡고 있습니다.

Netflix, Facebook, Instagram, Airbnb 등 다양한 글로벌 서비스에서 React를 활용하고 있으며, 풍부한 라이브러리와 커뮤니티를 기반으로 지속적으로 발전하고 있습니다.

또한 React Native를 통해 웹뿐만 아니라 모바일 애플리케이션 개발에도 활용할 수 있어 높은 확장성을 가지고 있습니다.

React는 단순히 화면을 만드는 기술이 아니라 컴포넌트 구조, 상태 관리, 데이터 흐름을 체계적으로 설계할 수 있도록 지원하는 프론트엔드 기술이라고 생각합니다.

특히 사용자와 상호작용하는 화면을 효율적으로 구현할 수 있다는 점에서 현대 웹 서비스 개발에 매우 적합한 기술이며, 앞으로도 꾸준히 학습하고 발전시키고 싶은 기술입니다.

실제 서비스 수준의 프로젝트를 개발하며 기능 구현뿐만 아니라 사용자 흐름과 서비스 완성도의 중요성을 배울 수 있었습니다.

Medical Script는 당뇨 환자의 혈당 관리와 건강 기록을 돕기 위해 개발 중인 모바일 헬스케어 서비스입니다.

사용자는 식단 사진 촬영, 혈당 기록, 투약 관리, 통계 확인 등의 기능을 통해 자신의 건강 상태를 보다 쉽게 관리할 수 있습니다.

프로젝트에서는 모바일 프론트엔드 개발을 담당하고 있으며 React Native를 활용하여 실제 사용자 중심의 화면을 구현하고 있습니다.

특히 고령층 사용자도 쉽게 사용할 수 있도록 큰 버튼과 직관적인 인터페이스를 구성하는 데 집중하였습니다.

또한 SQLite를 활용한 오프라인 데이터 저장, 푸시 알림 기능, 주간·월간 건강 통계 화면 등을 구현하며 실제 서비스 개발 과정을 경험하고 있습니다.

이 프로젝트를 통해 모바일 환경에서의 사용자 경험(UI/UX) 설계와 상태 관리, API 연동, 데이터 구조 설계 능력을 향상시킬 수 있었습니다.

현재도 지속적으로 기능을 개선하며 개발을 진행하고 있는 프로젝트입니다.
